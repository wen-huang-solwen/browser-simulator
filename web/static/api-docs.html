Reels Scraper API

Scrape video metadata (views, likes, comments) from Instagram, Facebook, TikTok, and YouTube profiles.

Base URL: http://your-server:8000
Scrape Videos Session Status Upload Session Download CSV Error Handling
POST /api/v1/scrape Scrape videos and return CSV

Scrape video/reel metadata from a social media profile and return results as a downloadable CSV file. The platform is auto-detected from the URL.

Request

Content-Type: multipart/form-data

ParameterTypeDescription
profile_link required string Profile URL or username. Supported formats:
https://www.instagram.com/username/reels/
https://www.facebook.com/pagename/reels/
https://www.tiktok.com/@username
https://www.youtube.com/@username
@username (defaults to Instagram)
max_reels optional integer Maximum number of videos to scrape. Range: 1–9999. Default: 50
auth_file optional file Playwright session JSON file for authentication. Required for private profiles or to avoid login prompts. Max size: 5 MB.

Response

Returns a CSV file download with the following columns:

ColumnTypeDescription
linkstringFull URL to the video/reel
idstringVideo shortcode / unique identifier
viewsintegerView count (may be empty if unavailable)
likesintegerLike count (may be empty if unavailable)
commentsintegerComment count (may be empty if unavailable)
scraped_atISO 8601UTC timestamp when the data was scraped

Example: curl

# Basic scrape (no auth)
curl -X POST http://localhost:8000/api/v1/scrape \
  -F "profile_link=https://www.instagram.com/halei.lawyer/reels/" \
  -F "max_reels=20" \
  -o results.csv

# With authentication file
curl -X POST http://localhost:8000/api/v1/scrape \
  -F "profile_link=https://www.instagram.com/halei.lawyer/reels/" \
  -F "max_reels=10" \
  -F "auth_file=@session.json" \
  -o results.csv

Example: Python

# pip install requests
import requests

resp = requests.post(
    "http://localhost:8000/api/v1/scrape",
    data={
        "profile_link": "https://www.instagram.com/halei.lawyer/reels/",
        "max_reels": "20",
    },
)

with open("results.csv", "wb") as f:
    f.write(resp.content)

# With auth file
resp = requests.post(
    "http://localhost:8000/api/v1/scrape",
    data={
        "profile_link": "https://www.instagram.com/halei.lawyer/reels/",
        "max_reels": "10",
    },
    files={
        "auth_file": open("session.json", "rb"),
    },
)

Example: JavaScript (fetch)

const form = new FormData();
form.append("profile_link", "https://www.instagram.com/halei.lawyer/reels/");
form.append("max_reels", "20");

const resp = await fetch("http://localhost:8000/api/v1/scrape", {
  method: "POST",
  body: form,
});

const blob = await resp.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url;
a.download = "results.csv";
a.click();

GET /api/session/status Check authentication status

Returns whether saved session files exist for each platform.

Example Request

curl http://localhost:8000/api/session/status

Example Response

{
  "instagram": true,
  "facebook": false,
  "tiktok": false
}
POST /api/session/upload Upload a session file

Upload a Playwright session JSON file for a specific platform. The server will restart its browser context to use the new session.

Request

Content-Type: multipart/form-data

ParameterTypeDescription
platform required string One of: instagram, facebook, tiktok
file required file Playwright session JSON file (must contain a cookies key). Max: 5 MB.

Example

curl -X POST http://localhost:8000/api/session/upload \
  -F "platform=instagram" \
  -F "file=@.auth/session.json"

Example Response

{
  "ok": true,
  "status": {
    "instagram": true,
    "facebook": false,
    "tiktok": false
  }
}
GET /api/download/{filename} Download a previously saved CSV

Download a CSV file that was previously generated by the web UI scrape. The filename is returned in the SSE scrape results.

Path Parameters

ParameterTypeDescription
filename required string CSV filename (e.g. halei.lawyer_reels_20260308_080233.csv)

Example

curl -O http://localhost:8000/api/download/halei.lawyer_reels_20260308_080233.csv
Error Handling

All error responses return JSON with a detail field:

{ "detail": "Cannot parse username from: !!!invalid!!!" }

Status Codes

CodeMeaning
200Success. CSV file returned (for scrape) or JSON returned (for other endpoints).
400Bad request. Invalid profile link, invalid auth file, or invalid platform.
404Not found. No videos/reels found for the profile, or download file does not exist.
422Validation error. Missing required fields or parameter out of range.
